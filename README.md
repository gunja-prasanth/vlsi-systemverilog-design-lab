## 📖 Project Overview  

This repository documents a **30-day intensive VLSI practice journey** focused on strengthening RTL design, verification methodology, timing awareness, power-conscious design, and system-level integration.

Unlike basic RTL exercises, this phase emphasizes:

- 🧠 Industrial thinking  
- 🧪 Verification-first mindset  
- ⏱ Timing & pipeline awareness  
- 🔋 Low-power considerations  
- 🔗 Subsystem-level integration  

Each day includes:
- RTL implementation  
- Testbench & simulation  
- Waveform verification  
- Documentation of design decisions  

---

# 📅 30-Day Breakdown

---

## 🧩 WEEK 1 – Industrial-Grade RTL Blocks  

Focus: Writing clean, reusable, parameterized RTL.

✔ Parameterized ALU (with flags Z, C, V)  
✔ Multi-mode counter (Up/Down/Load/Reset)  
✔ Universal shift register  
✔ Priority encoder & decoder  
✔ FSM-based traffic light controller  
✔ Clock divider & pulse generator  
✔ RTL refactoring & assertion cleanup  

📌 Outcome:  
Structured RTL with scalable design patterns.

---

## 🧪 WEEK 2 – Verification & Assertion Thinking  

Focus: Becoming verification-aware.

✔ Self-checking testbenches  
✔ Tasks & functions in the TB  
✔ Randomized stimulus testing  
✔ Assertion-based verification  
✔ Synchronous FIFO design  
✔ FIFO overflow/underflow testing  
✔ Verification documentation  

📌 Outcome:  
Designs validated with the structured verification methodology.

---

## ⏱ WEEK 3 – Timing & Power Awareness  

Focus: Thinking beyond simulation.

✔ Combinational vs pipelined adder comparison  
✔ Register retiming concepts  
✔ Clock gating implementation  
✔ Low-power FSM encoding comparison  
✔ Glitch-free MUX design  
✔ Sync vs Async reset strategies  
✔ Setup/Hold timing theory documentation  

📌 Outcome:  
Understanding of latency, throughput, and power optimization.

---

## 🔗 WEEK 4 – SoC & System-Level Blocks  

Focus: Integration & subsystem design.

✔ UART Transmitter (FSM-based)  
✔ UART Receiver  
✔ Loopback verification  
✔ Simple APB-like bus protocol  
✔ Round-robin arbiter  
✔ Interrupt controller  
✔ Mini subsystem integration  
✔ Bug injection & debug workflow  
✔ Final documentation & polish  

📌 Outcome:  
Subsystem-level design thinking aligned with real SoC architectures.
